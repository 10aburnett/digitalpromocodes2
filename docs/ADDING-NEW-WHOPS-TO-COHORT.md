# Adding New Whops to the Cohort - Step-by-Step Guide

**Last Updated:** 2026-01-08
**Current Cohort Size:** 121 (was 101, added 20)

---

## Prerequisites

Before adding new whops, ensure:
1. Content has been rewritten using the Claude workflow (export to `101 whop writing` repo)
2. Rewritten content JSON is copied back to this repo
3. Database records have been updated with new content

### 1. Export Content for Rewriting

### 2. Rewrite Content (External)

Use Claude workflow in the `101 whop writing` repo to rewrite content for all whops.

### 3. Import Rewritten Content

Copy rewritten JSON back to this repo:

### 4. Upsert Content to Both Databases


### 5. Set Indexing Status in Both Databases

**CRITICAL:** Set ALL FOUR fields correctly:
- `indexingStatus` = 'INDEX' (text field)
- `indexing` = 'INDEX' (enum field)
- `retired` = false
- `retirement` = null

```bash

// Use updateMany with:
// { indexingStatus: 'INDEX', indexing: 'INDEX', retired: false, retirement: null }
"
```

**Verify with:** `node scripts/verify-100-records.mjs`

### 6. Update Launch Cohort Configuration

Edit `src/lib/launch-cohort.ts`:
- Update comment: `(121 slugs)` → `(141 slugs)` (or new count)
- Add new slugs to `LAUNCH_COHORT_SLUGS` Set

```typescript

// Added: YYYY-MM-DD
'new-slug-1',
'new-slug-2',
// ... etc
```

### 7. Update Sitemap Script

Edit `scripts/build-sitemaps-clean.ts`:
- Update `EXPECTED_COHORT_COUNT` constant
- Update comment count
- Add new slugs to `LAUNCH_COHORT_SLUGS` array

### 8. Regenerate Internal Link Graph

```bash
npx ts-node scripts/generate-link-graph.ts
```

Then copy to src:
```bash
cp public/data/graph/neighbors.json src/data/graph/neighbors.json
```

### 9. Rebuild SEO Indexes

```bash
npx tsx scripts/build-seo-indexes.ts
```

Verify new slugs are NOT in NOINDEX_PATHS.

### 10. Rebuild Sitemaps

```bash
npx tsx scripts/build-sitemaps-clean.ts
```

Should show new count (e.g., "121 URLs").

### 11. Verify No Orphan Pages

```bash
node scripts/check-inbound-links.mjs
```

**Must show:**
- 0 orphan pages
- All new slugs have ≥1 inbound link

### 12. Run Build

```bash
npm run build
```

Ensure no errors.

### 13. Push to GitHub

```bash
git add -A
git commit -m "feat: Add [N] new whops to cohort (now [TOTAL])"
git push origin development
git checkout main && git merge development && git push origin main
git checkout development
```

### 14. Update Vercel Environment Variable

**MANUAL STEP in Vercel Dashboard:**
- Find cohort count env variable
- Update from old count to new count
- Redeploy

### 15. Post-Deploy Verification

1. Check a new page is accessible: `https://whoppromocodes.com/offer/[new-slug]`
2. Verify meta robots shows `index,follow` (not `noindex`)
3. Submit updated sitemap to Google Search Console

## Troubleshooting

**Pages showing noindex:**
- Check `indexingStatus` AND `indexing` fields in DB (both must be 'INDEX')
- Run `npx tsx scripts/build-seo-indexes.ts` to regenerate
- Verify slug not in `NOINDEX_PATHS`

**Orphan pages:**
- Run `node scripts/check-inbound-links.mjs`
- Regenerate link graph: `npx ts-node scripts/generate-link-graph.ts`

**Sitemap count wrong:**
- Check `EXPECTED_COHORT_COUNT` in `scripts/build-sitemaps-clean.ts`
- Check slug array has all entries

**Build fails:**
- Check for TypeScript errors in modified files
- Ensure all slugs exist in database
